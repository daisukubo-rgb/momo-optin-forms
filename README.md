# MoMo オプトインフォーム

メルマガ登録用オプトインフォーム（Xserver設置用）

## 設置先

Xserver: `momo-gpt.com/email-newsletter/` 配下

| パス | 内容 |
|------|------|
| `prospect/index.html` | 見込み客向けフォーム |
| `prospect/thanks.html` | 見込み客 登録完了ページ |
| `customer/index.html` | 既存顧客向けフォーム |
| `customer/thanks.html` | 既存顧客 登録完了ページ |
| `agent/index.html` | パートナー向けフォーム |
| `agent/thanks.html` | パートナー 登録完了ページ |

## フロー

1. ユーザーがフォーム入力 → JS確認オーバーレイ
2. GAS Web App（doPost）に送信
3. スプレッドシートに「承認済み」で登録
4. タイプ別の自動返信メール送信
5. 完了ページにリダイレクト

## 確認画面（confirm）が無い理由

従来のMyASP構成では `フォーム → 確認画面（別ページ） → 完了画面` の3ページ構成だったが、MyASP脱却にあたり以下の理由で確認画面を独立ページとしては廃止した。

- **確認機能はフォームページ内にJavaScriptオーバーレイとして統合済み**
  - 送信ボタン押下 → 入力内容を表示した確認ダイアログが表示 → 確認後に送信
  - ページ遷移なしで確認できるため、UXが向上
- **サーバーサイドの確認画面が不要**
  - MyASPではサーバー側で確認画面を生成していたが、GAS doPostはフォームデータを受け取って即処理するため、中間ページが不要
- **ページ数削減**により保守コストが下がる（9ファイル → 6ファイル）

元の確認画面ファイル（`*_confirm.html`）はNotionCRM-GASの `Opt-In/` フォルダにアーカイブとして残っている。

## デプロイ

Xserverの `/public_html/email-newsletter/` にこのリポジトリの内容をそのまま配置する。
